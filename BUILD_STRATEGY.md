# Build Strategy

## Orden recomendado de construcción
1. **Contracts first**: modelos de organización, licencias, dispositivos, sesiones, políticas.
2. **Desktop MVP core**: conexión SSH/SFTP, tabs/workspaces, vault local, settings seguros.
3. **Control Plane MVP**: auth, tenancy, subscriptions, licensing, sync metadata.
4. **AI MVP**: command explanation + chat operacional.
5. **Hardening**: auditoría, posture checks, offline activation flows.

## Razón estratégica
Este orden maximiza entrega de valor sin sacrificar seguridad ni introducir complejidad prematura.
